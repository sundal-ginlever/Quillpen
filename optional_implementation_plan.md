# Quillpen: 향후 고도화를 위한 선택적 구현 계획 (Optional)

이 문서는 Quillpen(inkcanvas) 프로젝트의 기술적 완성도를 높이고 기능을 확장하기 위한 차세대 작업 아이템들을 정리한 것입니다.

## 1. 아키텍처 현대화
### 1.1 Vite 기반 프로젝트 구조 전환
- **목표**: 단일 HTML 파일을 ES 모듈 단위로 분리하여 유지보수성 향상.
- **작업**:
  - `src/` 폴더 내에 `css`, `js`, `components` 분리.
  - 빌드 시 `vite-plugin-singlefile`을 사용하여 여전히 단일 HTML 출력을 유지.
  - TypeScript 도입을 통한 코드 타입 안정성 확보.

## 2. 사용자 경험(UX) 고도화
### 2.1 오프라인 모드 및 PWA 강화
- **목표**: 인터넷 연결이 불안정한 환경에서도 완벽한 작업 지원.
- **작업**:
  - Service Worker를 통한 모든 정적 자산(CSS/JS/WebFonts) 오프라인 캐싱.
  - IndexedDB를 활용한 대용량 로컬 데이터 저장 (LocalStorage 용량 한계 극복).
  - Background Sync를 통한 재연결 시 자동 업로드 엔진 구현.

### 2.2 실시간 동시 편집 (Realtime Collaboration)
- **목표**: 여러 사용자가 동시에 하나의 캔버스에서 작업.
- **작업**:
  - Supabase Realtime 채널을 이용한 위젯 위치/내용 실시간 브로드캐스팅.
  - 다른 사용자의 커서 위치 시각화.
  - 동시 수정 시 충돌 방지 로직 (Operational Transformation 또는 CRDT) 도입.

## 3. 기능 확장
### 3.1 템플릿 시스템
- **목표**: 마인드맵, 칸반 보드, 회의록 등 미리 정의된 위젯 배치 템플릿 제공.
### 3.2 이미지 OCR 및 스마트 스캔
- **목표**: 업로드된 이미지 내 텍스트를 인식하여 메모 위젯으로 변환.
### 3.3 화이트보드 레코딩 및 재생
- **목표**: 캔버스에서의 작업 과정을 녹화하고 단계별로 다시 보기 기능.

## 4. 분석 및 운영
### 4.1 성능 모니터링 도입
- **목표**: 실제 사용자의 렌더링 프레임(FPS) 및 로딩 시간 분석.
### 4.2 오류 추적
- **목표**: 클라이언트 측 런타임 에러 발생 시 자동 로깅 및 알림.
